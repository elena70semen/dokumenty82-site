# Rollback Runbook

## Scope

This runbook describes static-export rollback readiness. It is not a production rollback and does not approve public launch.

## Artifact versioning

Every reviewed build should be stored as a versioned static artifact:

- current build artifact path: `C:\new\dokumenty-dlya-biznesa\code\out`;
- local copy path: `C:\new\dokumenty82-p0-local`;
- previous build artifact placeholder: `<previous-versioned-static-export-path>`;
- evidence manifest: `C:\new\dokumenty-dlya-biznesa\code\evidence\staging-hosting-rollback\rollback\artifacts\current-static-export-manifest.json`.

## Restore previous static export

1. Confirm owner/ops rollback approval.
2. Stop writes/deploy activity.
3. Preserve current failed artifact as evidence.
4. Replace the active static root with `<previous-versioned-static-export-path>`.
5. Restart or reload the static server.
6. Validate public routes, sitemap and robots.

## Validate rollback locally

1. Copy the selected static artifact to `C:\new\dokumenty82-p0-local`.
2. Run local static server on `127.0.0.1:4173`.
3. Check HTTP 200 for all 14 P0 routes.
4. Check `/robots.txt` and `/sitemap.xml`.
5. Re-run browser/local evidence if the rollback candidate will be reviewed.

## Stop and restart local server

If a local preview process is running on `127.0.0.1:4173`, identify it before stopping:

```powershell
Get-NetTCPConnection -LocalAddress 127.0.0.1 -LocalPort 4173 -State Listen
```

For a future production server, use provider-approved service controls only. Do not use destructive commands against unknown paths.

## CDN/cache later

If CDN or object hosting is used later:

- purge only the approved site distribution;
- record purge request ID;
- re-check 14 P0 routes after purge;

## Approval

Rollback requires owner/ops approval. Legal/privacy review is required if the rollback changes policy text, data collection behavior or provider surface.

## Evidence to capture

- artifact version before rollback;
- artifact version after rollback;
- route HTTP proof;
- sitemap/robots proof;
- screenshot of critical routes when available;
- approval note;
- cache purge note if used.
